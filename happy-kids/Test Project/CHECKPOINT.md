## Что НЕ сделано (бэклог)
- i18n RU/EN и мультивалютность ₽/$ — переключатели в шапке пока статичны (без логики).
- Реальный поиск (сейчас иконка ведёт в /catalog).
- Стаб-страницы из ссылок: `/delivery`, `/about`, `/contacts`, `/catalog?view=series` — ещё нет (404).
- Реальные фото товаров (сейчас инлайн-SVG-кубики), реальный каталог/бэкенд, оплата.
- Шаг «view=series» на каталоге как отдельный режим витрины серий.
- Визуальная проверка в браузере (в окружении нет браузерных инструментов — проверял build+curl).

## Зафиксированные решения
1. B2C, цель — продажи. Сценарий — быстрая покупка.
2. Двойная аудитория: выбирает ребёнок 12 лет, **платит родитель**. Витрина/PDP — игровые, корзина/чекаут — спокойные.
3. Платформа — custom **Next.js 16** (этот репозиторий). Адаптив, равный приоритет устройств.
4. Каталог < 50 SKU, плоская структура. Товары **с сериями** (вариации).
5. Локали **RU/EN**, валюты **RUB/USD**.
6. Фильтры: возраст до 12+, детали ≤100, серия, сложность=простая, цена, бренд (уровня Lego).
7. **Account исключён из MVP** (бизнес-решение) — гостевой чекаут обязателен.
8. Бренд **Happy Kids**; визуал — комфортный для ребёнка (WCAG AA, «tween»).
9. UI System v1.0: палитра **комплементарная** оранжевый `#FF7A33` × синий `#2E7DF6`;
   шрифты **Comfortaa** (заголовки, замена Fredoka ради кириллицы) + **Nunito Sans**
   (содержание), обе OFL/свободные, обе с кириллицей.
